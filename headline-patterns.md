# 标题模板库

## 使用原则

标题优先体现：

1. 冲突
2. 变化
3. 行业意义
4. 技术对象

能写出观点，就不要只写信息。

## 常用模板

### 1. 旧范式失灵型

- "`[旧方法/旧范式]` 不够用了？`[论文/模型]` 想换个解法"
- "`[老问题]` 卡住行业太久，这篇论文从底层重写"

适合：

- 长上下文
- 推理
- Agent
- 训练效率

### 2. 结果先行型

- "`[关键结果]` 只是开始，`[论文名]` 真正想改的是`[更大问题]`"
- "`[数字/成绩]` 背后，藏着这篇论文真正的野心"

适合：

- 有明显 SOTA 或成本优势的论文

### 3. 产业冲击型

- "`[方向]` 又要变天了？`[团队]` 放出`[模型/论文名]`"
- "如果这篇论文成立，最先被改写的可能是`[场景/岗位/产品形态]`"

适合：

- 产品化潜力强
- 对工作流有影响

### 4. 认知反转型

- "大家都在卷`[A]`，这篇论文偏要从`[B]`下手"
- "你以为它在优化`[表层目标]`，其实它在重写`[底层目标]`"

适合：

- 方法直觉和主流做法明显不同

### 5. 保守传播型

- "`[论文名]` 解读：`[一句话亮点]`，但真正价值可能在`[点]`"
- "`[团队]` 新论文想解决`[问题]`，效果如何还要再看"

适合：

- 证据还不够硬
- 用户明确要求不要太标题党

## 标题生成流程

先写 5 个候选标题，再选最强的 1 个：

1. 一个结果型
2. 一个冲突型
3. 一个行业影响型
4. 一个保守型
5. 一个最像公众号的传播型

然后筛掉：

- 正文撑不住的
- 像论文翻译的
- 只有名词堆砌没有观点的

## 慎用标题信号

- "震惊"
- "炸裂"
- "史上最强"
- "完全颠覆"
- "人类已经不需要"

这些词比较廉价，有可能会让稿子失去可信度，仅允许在没有更多闪光点的时候使用。
